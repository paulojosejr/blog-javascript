O que significa o This?

Entendendo o This

O This é uma forma de referenciar o objeto atual de uma execuxão, não é um conceito tão simples de passar. O This acessa quem é o dono do código que está sendo executado naquele momento.

No JavaScript o This pode variar dependendo de como a função é chamada. Por exemplo:

Abrindo o browser e apertando f12 para abrir o console. Vamos criar e chamar uma função:

function f1() { console.log(this === window) }
f1()

Executando e chamando a função, vemos que é verdadeiro, ou seja, quando o this é usado nesse contexto de fato o This aponta para o objeto global do browser.

O This é sempre o objeto que está sendo referenciado naquele contexto de execução que é o dono daquela função que está sendo executada naquele contexto. O problema é que no JavaScript dependendo da forma que você chamar a função o This pode não apontar para o window mas para algo totalmente diferente.

Usando essa mesma função que foi criada e chamada e que de fato está apontando para o objeto global, vamos mudar a forma que ela está sendo chamada e vamos chamar onclick.

document.getElementsByTagName('body')[0].onclick = f1

Vamos usar o body e pegar o primeiro elemento da página e ao clicarmos na página o resultado será false.

Mas o resultado não era verdadeiro?

Sim porém o This varia dependendo da forma que a função está sendo chamada e esse sem dúvida é um dos motivos que mais causam confusão no JavaScript.

Vamos Criar uma nova função:

function f2() { console.log(this === document) }
f2()

Criamos uma função apontando para document e sabemos que no contexto global o This não aponta para document, será que quando clicar com o mouse o this irá apontar para document?

Agora vamos mudar a forma que chamamos a função para onclick.

document.getElementsByTagName ('body')[0].onclick = f2

Vemos que não é o document!

Vamos buscar o elemento no html com a tag body e associar a uma constante.

const body = document.getElementsByTagName ('body')[0]

Vamos criar novamente a função e verificar se é igual a constante que criamos.

function f2() { console.log(this === body) }

Depois que fizermos isso vamos chamar a função onclick.

document.getElementsByTagName ('body')[0].onclick = f2

Clicando vemos que o resultado é verdadeiro.

Quem de fato representa o this ao chamar a função F2 a partir do click? É o próprio elemento que está sendo clicado.

Porém se chamarmos a função f2...

f2()

Vamos ver que não é o body que está apontando para o This e sim o objeto window que mostramos anteriormente.

Por que arrow function?

Existiram dois grandes motivos para criar arrow function.

O primeiro motivo foi para reduzir a sintaxe, por exemplo:

const f3 = () => console.log(this === window)
f3()

O segundo motivo foi para ter um this que não varia, por exemplo:

document.getElementsByTagName ('body')[0].onclick = f3

Ao clicarmos na pagina vemos que o This não irá variar e se chamarmos a função também não irá váriar.

f3()

Com isso concluímos como funciona o This dentro de uma função e dentro de uma arrow function sabendo que dentro de uma função normal o This pode váriar dependendo de como ele é chamado. O This é definido no momento em que a função é definida, é o que chamamos de This Léxico.
